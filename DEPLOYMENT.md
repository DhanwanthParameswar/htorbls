## Overview

HTOR BLS manages:

- **Book catalog** (`booklist`) — IDs, titles, categories, notes, optional cover images
- **Active loans** (`librarylog`) — patrons, contact info, due dates, fines
- **Returns archive** (`libraryarchive`) — historical checkouts
- **Staff users** (`users`) — login with bcrypt-hashed passwords

The UI is server-rendered PHP with Bootstrap 5 (CDN), jQuery, and session-based authentication.

## Architecture

```
                    ┌─────────────────┐
                    │  Browser        │
                    └────────┬────────┘
                             │ HTTPS (optional: Cloudflare Tunnel)
                             ▼
                    ┌─────────────────┐
                    │  nginx          │  port 80 / 443 / 8090
                    │  + PHP-FPM      │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        *.php pages    config.local.php   images/books/
              │              │
              └──────┬───────┘
                     ▼
              ┌─────────────────┐
              │  MariaDB/MySQL  │  database: library
              └─────────────────┘
```

| Layer | Technology |
|-------|------------|
| Web server | nginx (Apache not required) |
| Application | PHP 8.1+ with `php-fpm` |
| PHP extensions | `mysql`, `gd`, `mbstring` |
| Database | MariaDB 10.6+ or MySQL 8+ |
| Auth | PHP sessions + `password_hash()` / `password_verify()` |

| Branch | Purpose | Deploy to |

- Demo-only UI/copy: commit only on `demo`.

## Requirements

### Minimum server (Linux)

- Ubuntu 22.04 LTS (or similar Debian-based distro)
- 1 GB RAM (2 GB+ comfortable with other services)
- nginx, PHP-FPM 8.1+, MariaDB/MySQL
- `git`, `openssl`, `curl`

### Packages (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install -y nginx php-fpm php-mysql php-gd php-mbstring mariadb-server git
```

### PHP-FPM socket

Ubuntu 22.04 typically uses:

```
unix:/run/php/php8.1-fpm.sock
```

Adjust `deploy/nginx-htorbls.conf` if your version differs (`php8.2-fpm`, etc.).

## Configuration

### `config.local.php` (required, not in git)

Create in the **repository root** (same directory as `index.php`):

```php
<?php
$username = "your_db_user";
$user_pass = "your_db_password";
```

`db_connect.php` loads this file and connects to:

- **Host:** `localhost`
- **Database:** `library`

### `config.demo.php` (demo branch only)

Defines `DEMO_MODE` and `demo_render_banner()`. Loaded automatically when present:

```php
if (file_exists(__DIR__ . '/config.demo.php')) {
    require_once __DIR__ . '/config.demo.php';
}
```

Do **not** deploy `config.demo.php` to a real production library if you do not want the demo banner.

### `deploy/.credentials` (server only, not in git)

Created by `deploy/deploy.sh`. Used by reseed scripts:

```
Database:
  name: library
  user: htorbls
  password: <random>
  host: localhost

Demo login:
  username: demo
  password: DemoPass123
```

## Database

### Schema

Applied by `deploy/schema.sql` or manually:

| Table | Purpose |
|-------|---------|
| `users` | Staff logins (`username`, bcrypt `password`) |
| `booklist` | Catalog (`bookId` unique) |
| `librarylog` | Active loans; `bookId` FK → `booklist` |
| `libraryarchive` | Returned items |

### Demo seed data

File: **`demo/seed.sql`**

- Truncates all tables and inserts sample books, loans, and archive rows.
- Does **not** include the `demo` user row — `deploy/reseed-demo-db.sh` inserts that with a fresh bcrypt hash.

Default demo login after reseed:

| Field | Value |
|-------|-------|
| Username | `demo` |
| Password | `DemoPass123` |

Override password when reseeding:

```bash
sudo HTOR_DEMO_PASSWORD='YourPass' /path/to/htorbls/deploy/reseed-demo-db.sh
```

### Manual schema + user setup

```bash
mysql -u root -p < deploy/schema.sql
mysql -u root -p library -e "INSERT INTO users (username, password) VALUES ('admin', '$(php -r "echo password_hash(\"yourpassword\", PASSWORD_DEFAULT);")');"
```

## Local development

### 1. Clone and branch

```bash
git clone https://github.com/DhanwanthParameswar/htorbls.git
cd htorbls
git checkout demo   # or main for production-like setup
```

### 2. Database

```bash
sudo mysql -e "CREATE DATABASE library CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
sudo mysql -e "CREATE USER 'htorbls'@'localhost' IDENTIFIED BY 'localdev';"
sudo mysql -e "GRANT ALL ON library.* TO 'htorbls'@'localhost'; FLUSH PRIVILEGES;"
mysql -u htorbls -plocaldev library < deploy/schema.sql
```

### 3. `config.local.php`

```php
<?php
$username = "htorbls";
$user_pass = "localdev";
```

### 4. Web root

Point nginx/Apache document root at the repo root, enable PHP-FPM, or use PHP’s built-in server (development only):

```bash
cd htorbls
php -S localhost:8080
# Open http://localhost:8080/login.php
```

### 5. Seed demo data (optional)

```bash
# Create deploy/.credentials matching your local DB, or run reseed after deploy.sh pattern
mysql -u htorbls -plocaldev library < demo/seed.sql
# Then insert demo user with reseed script or manually
```

### 6. Book cover uploads

Ensure the directory exists and is writable by the web server:

```bash
mkdir -p images/books
chmod 775 images/books
```

On the **demo** branch, `new_book.php` uses `/var/www/htorbls/images/books/` — adjust paths in `new_book.php` for your local document root if needed.

## Production deployment

For a **real library** (no demo reset, no demo banner):

1. Use branch **`main`**.
2. Do **not** install demo cron or run `reseed-demo-db.sh`.
3. Create `config.local.php` with production DB credentials.
4. Run `deploy/schema.sql` once; migrate data as needed.
5. Configure nginx/Apache with SSL for your domain.
6. Set `images/books/` permissions for uploads.

Example nginx `root`:

```nginx
root /var/www/htorbls;
index index.php;
location ~ \.php$ {
    include snippets/fastcgi-php.conf;
    fastcgi_pass unix:/run/php/php8.1-fpm.sock;
}
```

## Portfolio demo site

Reference deployment: **https://lms.dhanwanth.com** (OCI + Cloudflare Tunnel).

### Quick install (Ubuntu, empty server)

```bash
sudo git clone -b demo https://github.com/DhanwanthParameswar/htorbls.git /var/www/htorbls
cd /var/www/htorbls
sudo bash deploy/deploy.sh
```

This will:

1. Install PHP-FPM, MariaDB, git (if missing)
2. Create database `library` and user `htorbls` with a random password
3. Apply `deploy/schema.sql`
4. Write `config.local.php` and `deploy/.credentials`
5. Configure nginx on **port 8090** (see `deploy/nginx-htorbls.conf`)
6. Load `demo/seed.sql` via `reseed-demo-db.sh`

### Filesystem permissions (recommended)

```bash
sudo chown -R ubuntu:www-data /var/www/htorbls
sudo find /var/www/htorbls -type d -exec chmod 775 {} \;
sudo find /var/www/htorbls -type f -exec chmod 664 {} \;
sudo chmod 640 /var/www/htorbls/config.local.php
sudo chown www-data:www-data /var/www/htorbls/config.local.php
```

Git operations as `ubuntu`; PHP-FPM reads as `www-data`.

```bash
git config --global --add safe.directory /var/www/htorbls
```

### Public URL

See [Cloudflare Tunnel](#public-access-cloudflare-tunnel) or put nginx behind your own reverse proxy / TLS terminator.

## Deploy scripts reference

All scripts live in **`deploy/`**. Run with `sudo` unless noted.

| Script | Purpose |
|--------|---------|
| **`deploy.sh`** | Full first-time install: packages, DB, nginx :8090, credentials, seed |
| **`reseed-demo-db.sh`** | Truncate + reload `demo/seed.sql` + recreate `demo` user |
| **`fix-demo-site.sh`** | Restore `config.local.php`, sync MySQL password, reseed, fix permissions |
| **`switch-app-to-demo-branch.sh`** | `git pull` demo branch + reseed (repo already cloned) |
| **`migrate-from-htorbls-deploy.sh`** | One-time move of `.credentials` and cron from old `~/htorbls-deploy` |

Environment variables:

| Variable | Default | Used by |
|----------|---------|---------|
| `HTOR_DEMO_PASSWORD` | `DemoPass123` | `deploy.sh`, `reseed-demo-db.sh` |
| `APP_DIR` | Parent of `deploy/` | `deploy.sh`, fix/switch scripts |
| `SUDO_USER` | `ubuntu` | Git pull ownership in `deploy.sh` |

## Public access (Cloudflare Tunnel)

Example: expose port **8090** as `lms.yourdomain.com`.

### Cloudflare Zero Trust (token-managed tunnel)

1. Zero Trust → **Networks** → **Tunnels** → your tunnel
2. **Public Hostname** → Add  
   - Subdomain: `lms`  
   - Domain: `yourdomain.com`  
   - Service: `http://localhost:8090`
3. Ensure DNS is proxied (orange cloud)

### Config-file managed tunnel

Add to `~/.cloudflared/config.yml` **above** the catch-all `http_status:404` rule:

```yaml
  - hostname: lms.yourdomain.com
    service: http://localhost:8090
```

See `deploy/cloudflared-ingress-snippet.yml`.

Restart cloudflared after changes.

### Firewall

If using Cloudflare Tunnel only, you do **not** need to open port 8090 on your cloud provider security list. For direct access, allow **TCP 8090** (or terminate TLS on nginx 443).

## Daily database reset (demo)

Install cron (once):

```bash
sudo cp /var/www/htorbls/deploy/cron-demo-reseed /etc/cron.d/htorbls-demo-reseed
```

Default schedule: **04:00** server local time daily.

Log: `/var/log/htorbls-demo-reseed.log`

Manual reseed:

```bash
sudo /var/www/htorbls/deploy/reseed-demo-db.sh
```

On the server:

```bash
cd /var/www/htorbls
git pull origin demo    # or main for production
```

## Security notes

- **SQL injection:** Much of the legacy code interpolates user input into SQL. Do not expose production data to untrusted users without hardening. The demo site mitigates risk with daily reseeds and a single shared login.
- **Secrets:** Never commit `config.local.php` or `deploy/.credentials`.
- **Demo login:** Shared `demo` / `DemoPass123` is intentional for portfolios; change via `HTOR_DEMO_PASSWORD` or use a private demo.
- **Uploads:** Book images are stored as `images/books/{bookId}.jpeg`. Validate paths and permissions on production.
- **Sessions:** Default PHP session files; configure `session.cookie_secure` when using HTTPS only.

## Troubleshooting

### `FATAL ERROR ... Failed to connect to MySQL`

- Check `config.local.php` username/password
- Verify MariaDB is running: `systemctl status mariadb`
- Test: `mysql -u htorbls -p library`

### `Access denied for user 'htorbls'@'localhost'`

Password mismatch between `config.local.php` and MySQL:

```bash
DB_PASS=$(awk -F': ' '/^  password:/{print $2}' deploy/.credentials)
sudo mysql -e "ALTER USER 'htorbls'@'localhost' IDENTIFIED BY '${DB_PASS}'; FLUSH PRIVILEGES;"
```

Or run: `sudo deploy/fix-demo-site.sh`

### `Missing deploy/.credentials`

```bash
sudo cp /path/to/old/.credentials /var/www/htorbls/deploy/.credentials
sudo chmod 600 /var/www/htorbls/deploy/.credentials
```

Or re-run `sudo deploy/deploy.sh` (overwrites app if not careful — prefer `fix-demo-site.sh`).

### Git `dubious ownership`

```bash
git config --global --add safe.directory /var/www/htorbls
# Or run git as the repo owner, not root
```

### PHP pages download instead of execute

Enable PHP-FPM in nginx; check `fastcgi_pass` socket path.

### Blank page / 500

```bash
sudo tail -f /var/log/nginx/error.log
sudo journalctl -u php8.1-fpm -f
```

### Book upload fails

```bash
sudo mkdir -p /var/www/htorbls/images/books
sudo chown www-data:www-data /var/www/htorbls/images/books
sudo chmod 775 /var/www/htorbls/images/books
```

Confirm `new_book.php` upload path matches your install directory.

### Demo banner not showing

- On **`demo`** branch only: `config.demo.php` must exist
- Pages must call `demo_render_banner()` after `<body>` (included on demo branch pages)
